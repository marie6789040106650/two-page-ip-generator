# 修改前检查清单

在进行任何代码修改前，请确认：

## 影响范围评估
- [ ] 明确要修改的具体文件
- [ ] 识别可能受影响的其他文件
- [ ] 确认是否需要更新类型定义
- [ ] 检查是否有测试需要更新

## 兼容性检查
- [ ] 新修改是否破坏现有API
- [ ] 是否需要添加向后兼容支持
- [ ] 组件props是否保持兼容

## 修改策略
- [ ] 优先使用扩展而非修改
- [ ] 使用可选参数
- [ ] 保留deprecated标记而非直接删除

## 确认问题
请在修改前回答：
1. 这个修改的最小范围是什么？
2. 有哪些文件绝对不应该被修改？
3. 修改后如何验证功能正常？
